- **Compliance** - Meeting regulatory/policy requirements

---

## ğŸ“Š Level-by-Level Breakdown

### Developing (0.0-1.0)
- Security not considered in code
- Doesn't know OWASP top 10
- Asks "is this secure?" after the fact
- Treats security as external concern
- Learning security basics

### Competent (1.0-2.0)
- Follows security checklist items
- Avoids obvious vulnerabilities (SQL injection, etc.)
- Asks for security review before shipping
- Aware security matters but not proactive
- Learning threat modeling

### Proficient (2.0-3.0)
- Builds security in from the start
- Thinks through threat models
- Identifies security risks proactively
- Code exemplifies secure practices
- Helps others write secure code
- Compliance-minded

### Expert (3.0-3.5)
- Exceptionally security-aware
- Mentors others on threat modeling
- Identifies novel security risks
- Security decisions are sound
- Goes beyond compliance

### Authority (3.5-4.0)
- Org's security authority
- Sets security standards and culture
- Shapes security decisions org-wide
- Security reviews seek their input
- Drives security improvements
