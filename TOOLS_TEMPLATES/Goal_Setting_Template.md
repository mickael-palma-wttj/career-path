# Goal Setting Template

Ready-to-use template for creating development and career goals aligned with SMART principles and organizational objectives.

---

## How to Use This Template

**When to use:**
- Annual goal setting cycle
- Mid-year review updates
- New development plan creation
- Quarterly check-ins on goals
- Promotion readiness planning

**How to fill it out:**
1. Copy this template
2. Fill in each section with IC input
3. Manager reviews and provides feedback
4. Refine together
5. Save for reference and quarterly check-ins

**Tip:** Start with big picture (what do we want to accomplish?), then break down into specific goals.

## 4. GOAL ASSESSMENT FRAMEWORK (Is this a good goal?)

Ask yourself:

- ✓ Can I describe it clearly to someone else?
- ✓ Will I know when it's complete?
- ✓ Does it matter to the business?
- ✓ Does it help me grow?
- ✓ Is it achievable (not impossible)?
- ✓ Can I make progress on it this period?
- ✓ Does it connect to my career vision?
- ✓ Will I be excited about completing it?

*If you said yes to 6+ of these, you've got a solid goal.*

## 8. APPENDIX: SMART GOAL EXAMPLES

### ❌ Weak Goal (not SMART)
```
"Get better at public speaking"

Problems:
- Not specific (what kind?)
- Not measurable (how do you know?)
- Not time-bound (when?)
```

### ✅ Strong Goal (SMART)
```
"Give 2 talks at internal tech meetings by December, 
focusing on [technical topic]"

- Specific: 2 talks on specific topic
- Measurable: 2 talks completed, feedback collected
- Achievable: Regular talks happen monthly
- Relevant: Leadership skill for next level
- Time-bound: By December
```

---

### ❌ Weak Goal
```
"Improve code quality"

Problems:
- Too vague
- Hard to measure
- No timeline
```

### ✅ Strong Goal
```
"Increase test coverage for [system] from 65% to 85% by end of Q2"

- Specific: Test coverage for specific system
- Measurable: 85% test coverage (vs. 65% baseline)
- Achievable: Clear path with focus
- Relevant: Impacts system reliability
- Time-bound: End of Q2
```

---

### ❌ Weak Goal
```
"Mentor junior developers"

Problems:
- No specificity
- No measure of success
- Vague timeframe
```

### ✅ Strong Goal
```
"Formally mentor 2 junior developers, meeting 
bi-weekly, through end of year. Success measured 
by their promotion to Confirmed level."

- Specific: 2 junior devs, bi-weekly meetings
- Measurable: Promotion readiness achieved
- Achievable: With time allocation
- Relevant: Leadership skill for Senior level
- Time-bound: Through end of year
```
